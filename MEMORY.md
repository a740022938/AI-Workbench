# MEMORY.md - 长期记忆

## 项目记忆

### AI Vision Data and Training Workbench
**状态**: v0.1.0-preview 已发布
**技术栈**: Python, PyQt5, Ultralytics YOLO
**核心特点**:
- OpenClaw 协同工作流（核心差异化）
- 重复图暂存+恢复机制
- 轻量本地优先设计
- 标注减负思维

**市场定位**:
- YOLO Quick Tool: 功能简单
- Myolotrain: 太重（需 PostgreSQL+Docker）
- YoloEase: 依赖 CVAT
- LabelPixmap: 纯标注工具

## 架构演进 (2026-03-27 里程碑)

### 从单体架构到任务级协作架构
**已完成的核心架构**:
1. **运行期总账本**: `WorkbenchContext` 集中管理状态
2. **兼容属性层**: 保持旧代码可用
3. **主板插槽**: 为组织架构预留接口
4. **OpenClaw分身接口**: 6个高频动作的统一调用入口
5. **背包系统**: 角色专属的轻量存储（老板背包 + OpenClaw背包）
6. **状态查询系统**: 远程获取老板和OpenClaw状态（Telegram命令）
7. **操作决策规范**: 动作风险评估和执行策略（风险等级 + 执行模式）
8. **分身任务中心**: 完整的任务创建、执行、管理能力

### 架构分层
- **数据层**: `WorkbenchContext` + 背包系统 + 任务中心
- **业务层**: 现有方法 + 策略评估 + 任务执行
- **接口层**: 动作调用 + 状态查询 + 任务管理
- **扩展层**: 任务模板 + 策略规则 + 背包扩展

### 安全施工文化
- **备份机制**: 每次修改前自动备份
- **测试流程**: `auto_test.py` 验证框架结构

**第十三期 - 历史实验分组与排序**：

**第十四期 - 历史实验批量操作**：
- 批量操作对话框 (`ui/batch_operations_dialog.py`)：完整的前台界面

- 批量导出：支持TXT、JSON、CSV三种格式的实验摘要导出

### 质检中心第一期 - 标注异常检查最小闭环

**异常检查类型**：
1. 空标签文件 (`EMPTY_LABEL`)
2. 类别ID越界 (`CLASS_ID_OUT_OF_RANGE`)
3. 标签行格式错误 (`INVALID_LABEL_FORMAT`)
4. 框坐标越界 (`BOUNDING_BOX_OUT_OF_BOUNDS`)
5. 框宽高<=0 (`INVALID_BOUNDING_BOX`)
6. 重复框 (`DUPLICATE_BOUNDING_BOX`)

**质检工作流**：
- 完整质检界面：问题列表、筛选、排序、详情展示
- 双击跳转：从问题直接定位到对应图片
- 实时刷新：修改后立即验证修复效果
- 架构分层：核心业务逻辑独立，界面层专注展示

**第十期 - 按图片修复后差异查看**：

- 审计跟踪：为图片级修复审计预留接口，支持查看详细修复回执

**第十一期 - 按图片人工处理入口**：
- 人工处理识别：在图片详情中突出显示需人工处理的问题
- 问题类型概览：统计需人工处理的问题类型和数量
- 一键处理入口：从图片汇总视图直接切换到问题列表查看人工处理问题

3. **扩展性预留**：为其他训练器预留通用接口，不为小场景写死

5. **用户工作流完整**：从发现问题到修复验证的完整闭环

### 核心设计原则

1. **连锁集团思维**：预留升级口、扩展口、替换口、分店口
2. **主板层减负**：MainWindow持续瘦身，不包办所有业务
3. **YOLO全流程作战平台**：不只是标注工具，而是AI可调度的专业工作台
4. **通用化设计**：训练中心不按YOLO写死，走"通用训练中心+模型适配器体系"
5. **OpenClaw友好**：持续补动作词典、状态快照、动作回执、任务编排、回滚等机制
