# Support

## Best Ways to Get Help

### 1. Read the docs first
Please check the following before opening an issue:
- `README.md`
- `docs/WIN11_INSTALL_GUIDE.md`
- `docs/QUICK_START_WIN11.md`
- `docs/TROUBLESHOOTING.md`
- `docs/TECHNICAL_PREVIEW_SCOPE.md`
- `docs/KNOWN_ISSUES.md`

### 2. Open a GitHub issue
Use the issue templates for:
- bug reports
- feature requests
- documentation mismatches
- installation/runtime problems

### 3. Provide useful context
When asking for help, include:
- operating system
- Python version
- whether you used a virtual environment
- install command used
- exact error message
- screenshots or logs if relevant

## What This Repository Supports Best Right Now
At the current stage, support is strongest for:
- public repository questions
- installation/documentation issues
- startup/runtime bug reports
- architecture and workflow feedback

## What May Take Longer
Because this is a technical preview, responses may take longer for:
- environment-specific edge cases
- platform-specific rendering differences
- custom enterprise-style workflow requests
- unsupported third-party integration scenarios

## Security Issues
Please do not report suspected vulnerabilities publicly in a normal issue.
Follow the guidance in `SECURITY.md`.
