# Technical Preview Scope

This document clarifies the current public release position of **AI Workbench v1.0.0-stage**.

## Release Position

This repository is currently published as a **Technical Preview / Stage Public Development Release**.

It is intended to show:
- the main architecture direction
- the connected workflow across the major centers
- the current desktop application foundation
- the project’s extension direction for future growth

It should **not** be interpreted as a fully polished consumer-grade release.

## Best-Fit Users

This stage is best suited for:
- developers
- contributors
- architecture reviewers
- AI/ML users comfortable with Python environments

## What You Should Expect

At this stage, users should expect:
- manual environment setup
- dependency troubleshooting when needed
- occasional launch or platform-specific friction
- continued refinement of docs and runtime experience

## What Is Already Valuable

The current release is already useful for evaluating:
- project architecture
- module boundaries
- workflow coverage
- future extension points
- overall product direction

## Current Focus Areas

The highest-priority improvement areas are:
1. installation reliability
2. dependency clarity
3. startup consistency
4. public-facing documentation accuracy
5. cross-platform validation

## Non-Goals for This Stage

This stage does not promise:
- one-click installation
- perfect cross-platform parity
- enterprise collaboration features
- finished plugin marketplace flows
- fully polished end-user onboarding

## How to Read the Repository

When evaluating the repository, treat it as:
- a serious public development snapshot
- a collaboration-ready architecture base
- a product foundation still being tightened for real-world onboarding

## Feedback That Helps Most

The most valuable feedback right now is:
- installation failures
- dependency gaps
- startup/runtime errors
- documentation mismatches
- workflow friction in real usage
